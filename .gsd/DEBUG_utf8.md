# Debug Session: LangGraph Recursion Limit in Validation Agent

## Symptom
The `uvicorn` process throws a `RecursionError` generated by LangGraph: "Recursion limit of 25 reached without hitting a stop condition."
When looking at the dashboard agent logs, the agent repeatedly executes similar SQL queries checking for email vulnerabilities and fails to output a `<REPORT>` or parse JSON correctly. Once it hits an error like "System Error: Could not parse query JSON block...", it keeps retrying to execute tools in an infinite loop without ever wrapping up the phase until the graph aborts after 25 internal steps.

**When:** Running the "schema only" validation mode on `customers`.
**Expected:** The schema agent runs a few checks, outputs `<REPORT>...</REPORT>`, and the graph gracefully exits to `END`.
**Actual:** The node loops between tool generation and tool execution endlessly.

## Current Focus
hypothesis: The condition to exit the agent loop relies on finding `<REPORT>` in the AI's standard text output. If the agent gets confused and just keeps returning `{"action": "execute", ...}` without ever yielding `<REPORT>`, the `should_continue` edge constantly routes back to the tool node or agent node. Alternatively, `action: execute` might not be handled gracefully causing a parse error and retry loop.
Notice that the agent initially generated `{"action": "execute_query", "query": "...", "query_type": "sql"}`, but later generated `{"action": "execute", "query": "..."}` which might be invalid.
test: Check `data_quality_agent.py` routing logic for `should_continue`, `execute_tool`, and how it parses the LLM output.
expecting: We'll see that the agent gets stuck in a `agent -> tool -> agent` loop because it keeps failing to parse or the tool keeps returning an error that the agent keeps retrying on.
next_action: Analyze `backend/app/agents/data_quality_agent.py`.
## Fix Implemented
The root cause of the Recursion Limit was traced to pp/agents/state.py. 
The exploration_steps and alidation_steps counters were perfectly incremented and returned by the agent nodes. However, because these fields were **missing from the AgentState TypedDict definition**, LangGraph was silently dropping them during state updates. 
As a result, each loop read state.get('exploration_steps', 0) which was always 0. The max iterations check exploration_steps >= MAX_EXPLORATION_STEPS was never triggered.
The fields exploration_steps: int and alidation_steps: int were added to AgentState in pp/agents/state.py. This ensures LangGraph accurately increments the loops and terminates gracefully when retry limits are reached, completely resolving the Recursion limit of 25 reached error.
